Na tag <"input"> com atributo type="text", para que serve o atributo "placeholder"?
O atributo placeholder exibe um texto temporário dentro do campo de entrada que desaparece quando o usuário começa a digitar. Serve para dar uma dica ou instrução sobre o que deve ser inserido no campo.

Na tag <"input"> com atributo type="radio" ou type="checkbox", para que serve o atributo "checked"?
O atributo checked define o estado inicial do rádio ou checkbox como selecionado (marcado). Quando presente, o botão de rádio ou checkbox estará marcado quando a página carregar.

Na tag <"input"> para que serve o atributo "value"?
O atributo value define o valor inicial do campo de entrada. Este valor é o que será enviado ao servidor quando o formulário for submetido.

Que atributo deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez?
O atributo name deve ser utilizado para agrupar vários botões de rádio. Botões de rádio com o mesmo valor no atributo name fazem parte do mesmo grupo, permitindo que apenas um botão seja selecionado por vez.

Para que serve a tag <"label">? Para que serve o atributo "for" nessa tag?
A tag <"label"> é usada para definir um rótulo para um elemento de formulário, geralmente um <"input">. O atributo for vincula o rótulo a um elemento de formulário específico, cujo id corresponde ao valor do for. Isso melhora a acessibilidade e a usabilidade, pois clicar no rótulo foca o elemento associado.

Na tag <"option"> para que serve o atributo "selected"?
O atributo selected define a opção como selecionada quando a página é carregada. No contexto de um menu suspenso (<"select">), ele determina a opção que será mostrada como escolhida inicialmente.

Nas tags de formulário para que serve o atributo "required"?
O atributo required indica que o campo de entrada deve ser preenchido antes de o formulário ser enviado. Ele força o usuário a fornecer um valor válido para aquele campo.

Para que serve a tag <"textarea">?
A tag <"textarea"> é usada para criar uma área de texto multilinhas onde o usuário pode inserir grandes quantidades de texto. Diferente de <"input type="text""">, <"textarea"> permite entradas de múltiplas linhas.

Na tag <"form"> para que serve o atributo "action"?
O atributo action especifica a URL para onde os dados do formulário serão enviados quando ele for submetido.

Na tag <"form"> para que serve o atributo "method"?
O atributo method define o método HTTP usado ao enviar os dados do formulário. Os valores comuns são GET e POST.

Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, qual deve ser o valor do atributo "method" na tag <"form">?
O valor do atributo method deve ser POST. Diferente de GET, POST envia os dados do formulário no corpo da solicitação HTTP, não na URL.

Qual valor deve ser informado no atributo "method" na tag <"form"> em um formulário que será usado para fazer upload de arquivos?
O atributo method deve ser POST. Além disso, é necessário adicionar o atributo enctype="multipart/form-data" para permitir o upload de arquivos.

O que as tags <"fieldset"> e <"legend"> produzem?
A tag <"fieldset"> agrupa elementos relacionados dentro de um formulário, desenhando uma caixa ao redor do conteúdo. A tag <"legend"> define uma legenda para o grupo de elementos dentro do <"fieldset">, que geralmente aparece como um título dentro da caixa do <"fieldset">.
